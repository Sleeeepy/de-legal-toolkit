# Phase 3 — Legal document checklists

**Goal:** Verify the four legal documents against their statutory checklists, line by line.

Files (project-specific paths — discover them):
- Impressum: `src/app/**/impressum/page.tsx` or equivalent
- Datenschutzerklärung: `src/app/**/datenschutz/page.tsx` (versioned: `datenschutz-v1`, etc.)
- AGB: `src/app/**/agb/page.tsx`
- Widerrufsbelehrung (only if B2C consumer e-commerce): `src/app/**/widerrufsbelehrung/page.tsx`

## 3a. Impressum — § 5 DDG + § 18 Abs. 2 MStV

**Note**: § 5 TMG was replaced by § 5 DDG in February 2024. Check the header.

- [ ] Header reads **"Angaben gemäß § 5 DDG"** (not "§ 5 TMG"). FAIL if stale citation.
- [ ] Full name + Rechtsform stated (or implicit for Einzelunternehmer — natural-person address is sufficient)
- [ ] Ladungsfähige Anschrift (no PO box)
- [ ] E-Mail required; phone OR contact form (DDG accepts contact form as equivalent)
- [ ] Handelsregister + HRB/HRA-Nummer if applicable; else N/A noted
- [ ] USt-IdNr OR explicit Kleinunternehmer statement under § 19 UStG
- [ ] Wirtschafts-Identifikationsnummer (when its issuance phase-in is active)
- [ ] Aufsichtsbehörde for regulated professions; else N/A
- [ ] Berufsbezeichnung + berufsrechtliche Regelungen if regulated profession
- [ ] § 18 Abs. 2 MStV "Verantwortlich für den Inhalt" — required if editorial content exists (blog, magazine, news)
- [ ] OS-Plattform link (Art. 14 ODR-VO) — required if any B2C online transaction takes place
- [ ] § 36 VSBG Hinweis on Verbraucherschlichtungsstelle participation

The audit flags missing elements with the relevant citation; drafting the actual text from this checklist is a 20-30 minute manual task per Rechtsform.

## 3b. Datenschutzerklärung — Art. 13/14 DSGVO

- [ ] Identity + contact of controller (Art. 13(1)(a))
- [ ] DSB (DPO) contact if appointed, or correctly omitted if not (verify § 38 BDSG threshold: 20+ employees processing personal data automatically)
- [ ] Purposes + legal basis enumerated for each processing activity
- [ ] Legitimate interests cited where Art. 6(1)(f) used — balancing test on file
- [ ] Recipients / categories — cross-check against sub-processors.ts in Phase 4
- [ ] Third-country transfers + safeguards (SCCs / DPF named per processor)
- [ ] Storage duration / criteria (Art. 13(2)(a))
- [ ] Art. 15-22 rights enumerated
- [ ] Right to withdraw consent (Art. 13(2)(c)) — for every consent kind in the codebase
- [ ] Right to lodge complaint with **the competent state LfDI** — name the specific authority (see `legal-research/references/lfdi-by-bundesland.md`); generic "die zuständige Datenschutzbehörde" FAIL
- [ ] Automated decision-making (Art. 13(2)(f)) — disclosed if applicable
- [ ] Art. 14 source-of-data disclosure if autogenerated/scraped profiles exist
- [ ] Cookie / § 25 TDDDG storage section — strict-necessary vs consent-required distinguished
- [ ] Joint-controller (Art. 26) essence — if joint controllership exists

**Special-category data (Art. 9)**: if any form collects health/pregnancy/political/biometric data, the legal basis must be Art. 9(2)(a) explicit consent (or specific § 9 BDSG exception), not Art. 6(1)(b) contract.

## 3c. AGB — § 305-310 BGB

- [ ] User-group scope clear (visitor, customer, paying subscriber, child via parent)
- [ ] No surprising clauses (§ 305c)
- [ ] No unfair clauses per § 307-309 catalogue
- [ ] Liability limitations conform to § 309 Nr. 7 (no exclusion of intent / gross negligence / personal injury / mandatory product liability)
- [ ] Pricing transparency (VAT inclusion stated)
- [ ] Termination/Kündigung provisions
- [ ] Forum + governing law (Brussels Ia / Rome I compliant for consumers)
- [ ] **§ 312k Kündigungsbutton** — if subscription product exists, see Phase 7
- [ ] **§ 312j Bestellbutton** — if B2C checkout exists, see Phase 7

## 3d. Widerrufsbelehrung — Art. 246a EGBGB / §§ 355-356 BGB

Only applies for B2C consumer e-commerce contracts.

- [ ] Muster-Widerrufsbelehrung matches the official template (BGB Anlage 1)
- [ ] Muster-Widerrufsformular present (BGB Anlage 2)
- [ ] 14-day period stated
- [ ] Withdrawal address + email present
- [ ] **§ 356(4) BGB**: for services rendered before the 14 days expire — explicit consumer consent to early performance AND acknowledgement that withdrawal right expires must be obtained, and stated in the Widerrufsbelehrung
- [ ] **Electronic Widerrufsbutton (in force 19 Jun 2026)**: required for B2C distance contracts to offer an in-product electronic withdrawal route mirroring § 312k Kündigungsbutton. See Phase 7.

## Cross-document checks

- [ ] Operator identity consistent across all four docs
- [ ] Sub-processor list in Datenschutz matches `sub-processors.ts` (Phase 4)
- [ ] Consent-kinds enumerated in Datenschutz match the consent-kinds enum in the codebase
- [ ] No stale § 5 TMG / § 25 TTDSG citations anywhere (replaced Feb 2024 by DDG / TDDDG)

## Output schema

Per sub-section (3a / 3b / 3c / 3d), produce a checklist with each item flagged `[OK]` / `[MISSING]` / `[PARTIAL]` / `[N/A]` and a brief quote from the doc backing each finding. End each subsection with "Top 3 issues". Overall section ends with a single-paragraph verdict.

## Close gate

Lifted verbatim by the sink into each phase-3 issue's "Close gate" section. The bullets vary by sub-section (3a / 3b / 3c / 3d) — the sink lifts the bullets matching the failing sub-section, not the entire phase-3 set.

### Common to all sub-sections

- [ ] Re-audit of phase 3 (specifically the failing sub-section) with the toolkit at HEAD: this finding no longer detected.
- [ ] Every [MISSING] / [PARTIAL] item from the prior audit report is now [OK], with the verbatim quote from the doc captured in the re-audit log.

### 3a. Impressum

- [ ] Header reads "Angaben gemäß § 5 DDG"; zero remaining "§ 5 TMG" references anywhere in `src/`.
- [ ] All applicable checklist items above [OK]; N/A items annotated with reason.

### 3b. Datenschutz

- [ ] Right-to-complaint section names the specific LfDI (cross-checked against `legal-research/references/lfdi-by-bundesland.md`); generic "die zuständige Datenschutzbehörde" replaced.
- [ ] Zero remaining "§ 25 TTDSG" references; all replaced by "§ 25 TDDDG".
- [ ] Sub-processor list in the doc matches `sub-processors.ts` 1:1 (cross-check from Phase 4).
- [ ] Every Art. 6(1)(f) basis has its balancing test referenced (link or appendix).
- [ ] Special-category data (Art. 9) processing, if any, uses Art. 9(2)(a) explicit consent, not Art. 6(1)(b).

### 3c. AGB

- [ ] Every § 309 Nr. 7 BGB unfair-clause-catalogue item reviewed and conformed.
- [ ] Liability clauses do not exclude intent / gross negligence / personal injury / product liability.
- [ ] If subscription product exists: § 312k Kündigungsbutton clause present (also gated by Phase 7b).
- [ ] If B2C checkout exists: § 312j Bestellbutton clause + Art. 246a EGBGB pre-contractual info clause present (also gated by Phase 7a).

### 3d. Widerrufsbelehrung

- [ ] Muster-Widerrufsbelehrung and Muster-Widerrufsformular conform to BGB Anlagen 1 + 2 (verbatim or with documented justified deviations).
- [ ] § 356(4) BGB early-performance carve-out explicitly stated if services are rendered before the 14-day period expires.
- [ ] Post-19.06.2026: electronic Widerrufsbutton implemented (cross-check Phase 7c).

### Regression guard

For HIGH-severity phase-3 findings (stale statutory header, missing LfDI, missing § 309 Nr. 7 conformity):

- [ ] Repo-level grep in CI that fails the build on `§ 5 TMG` / `§ 25 TTDSG` / generic-LfDI strings appearing in any legal doc page. Name the script in the close comment.
